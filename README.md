# Цифровий гід першокурсника • ФМІФ УДУ

Мобільний веб-додаток для першокурсників факультету математики, інформатики та фізики Українського державного університету імені Михайла Драгоманова.

## Що змінилось у цій версії

- демо-дані замінені на реальні й наближені до реальних сценарії УДУ;
- додано окремий блок `Сервіси` з питаннями студентського, оплати, стипендій, довідок і заяв на пропуск занять;
- оновлено дизайн: виразніший hero-блок, сучасні картки, фільтри, прогрес першого тижня, інтерактивний FAQ;
- SQL-схема розширена під сервіси, кроки інструкцій і офіційні ресурси.

## Як запустити

1. Для звичайного перегляду можна відкрити `index.html` у браузері.
2. Для перевірки саме PWA-функцій запусти локальний сервер:
   `powershell -ExecutionPolicy Bypass -File .\\serve.ps1`
3. Після запуску відкрий `http://localhost:4173/`.
4. Для мобільної перевірки відкрий DevTools і ввімкни device mode.
5. Для реального встановлення на смартфоні сайт треба відкрити через HTTPS. Найпростіший варіант для цього проєкту - GitHub Pages.

## Структура

- `index.html` - розмітка застосунку.
- `styles.css` - адаптивний мобільний інтерфейс.
- `app.js` - дані, логіка чекліста, фільтрів і рендеру секцій.
- `pwa.js` - реєстрація service worker, install prompt і банери стану.
- `manifest.json` - конфігурація PWA.
- `service-worker.js` - кешування і офлайн-поведінка.
- `offline.html` - сторінка для офлайн-фолбеку.
- `serve.ps1` - простий локальний сервер для перевірки PWA через `localhost`.
- `db/schema.sql` - схема БД під контакти, ресурси, сервіси та FAQ.
- `docs/sources.md` - список офіційних джерел і примітки щодо точності.

## Важлива примітка

Не всі університетські сторінки однаково деталізують алгоритми саме для ФМІФ. Там, де точна факультетська сторінка не знайшлась, я позначив у застосунку, що сценарій узагальнений з офіційних сторінок УДУ інших факультетів і його треба уточнювати в деканаті.

## PWA-поведінка

- `manifest.json` додає іконки, кольори теми та режим `standalone`.
- `service-worker.js` кешує основні файли застосунку й показує `offline.html`, якщо немає мережі.
- Кнопка `Завантажити на смартфон` лишається в інтерфейсі й підказує правильний сценарій: `beforeinstallprompt` на Android/Chrome, інструкцію для iPhone або підказку про HTTPS/GitHub Pages.
- На `file://` service worker не працює, тому для офлайн-режиму й встановлення треба запускати застосунок через `localhost` або HTTPS.

## Публікація

- Якщо хочеш, щоб додаток встановлювався на телефон постійно, розмісти його на GitHub Pages.
